Decimal Variables

We have stored text and whole numbers. Now, we must store high-precision mathematics. Let's create a variable to hold a Decimal value, specifically, a Warp Factor of 9.97.

In computer science, decimal memory is called a Floating Point (or float). If it requires twice as much memory for extreme precision, it is called a double. Notice how system-level languages force you to explicitly declare this complex memory type, while scripting languages handle the translation automatically.

JavaScript (HTML / Browser)

JavaScript makes math incredibly simple. Under the hood, JavaScript treats all numbers as high-precision doubles automatically. We just use our standard let keyword. 

<!DOCTYPE html>
<html>
<head>
    <script>
        let warpFactor = 9.97;
        console.log(warpFactor);
    </script>
</head>
<body>
</body>
</html>

JavaScript (Node.js)

The Node.js environment acts identically. Assign the decimal to a variable, and the engine allocates the high-precision memory automatically. 

let warpFactor = 9.97;
console.log(warpFactor);

Python

Python detects the decimal point during the assignment and automatically classifies the variable as a floating-point number in its memory banks. 

warpFactor = 9.97
print(warpFactor)

C

C requires absolute precision. We cannot use int because 9.97 is not a whole number. We must explicitly use the float or double keyword to reserve decimal memory, and we must retrieve it using the %f placeholder. 

#include <stdio.h>

int main() 
{
    // Explicitly declare the variable as a Double-Precision Float
    double warpFactor = 9.97;
    
    // Retrieve using the floating-point placeholder
    printf("%f\n", warpFactor);
    return 0;
}

C++

C++ utilizes the exact same double keyword to reserve the memory block. However, the modern std::cout stream automatically handles the output formatting, saving you from having to type %f. 

#include <iostream>

int main() 
{
    // Explicitly declare the variable as a Double-Precision Float
    double warpFactor = 9.97;
    
    std::cout << warpFactor << std::endl;
    return 0;
}

Rust

Rust gives you the best of both worlds. You use the simple let keyword, but the Rust compiler strictly enforces it as an f64 (64-bit float) in memory to ensure maximum engine stability. 

fn main() 
{
    let warpFactor = 9.97;
    
    println!("{}", warpFactor);
}

Engineering Summary

Mixing up memory types is a catastrophic error in system languages. If you declare int warpFactor = 9.97; in C or C++, the compiler will literally chop off the .97 and store it as exactly 9, fundamentally destroying your trajectory data. You must always use float or double for decimal data.
