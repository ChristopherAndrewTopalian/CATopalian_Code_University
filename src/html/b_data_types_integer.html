Integers

In our first module, we transmitted a sequence of text (a String). But computers are fundamentally mathematical machines. Today, we will transmit a raw Integer, a whole number without a decimal point. Let's send the number 42 across our Comm-Link.

You will notice a major difference here. Scripting languages will understand the number automatically. System-level languages (like C and Rust) require strict formatting "placeholders" to explicitly tell the engine that a number is coming.

JavaScript (HTML / Browser)

JavaScript is dynamically typed. It recognizes that 42 is a number automatically, without quotes. 

<!DOCTYPE html>
<html>
<head>
    <script>
        console.log(42);
    </script>
</head>
<body>
</body>
</html>

JavaScript (Node.js)

The exact same automatic numeric recognition applies to the desktop engine. 

console.log(42);

Python

Python also handles raw numbers seamlessly. Just drop the integer right into the print function. 

print(42)

C

Here is where true systems engineering begins. In C, printf cannot just print a number directly. You must give it a String containing a placeholder, and then pass the number afterward. We use %d to tell the C compiler: "Expect a decimal integer here." 

#include <stdio.h>

int main() 
{
    // %d is the placeholder for an integer
    printf("%d\n", 42);
    return 0;
}

C++

C++ makes printing numbers much easier than C. The cout stream automatically detects that 42 is an integer and formats it for you. 

#include <iostream>

int main() 
{
    std::cout << 42 << std::endl;
    return 0;
}

Rust

Rust requires strict formatting similar to C, but uses a more modern placeholder. We use empty curly brackets {} to tell the Rust macro where to inject the number. 

fn main() 
{
    // {} acts as the target zone for the integer
    println!("{}", 42);
}

Engineering Summary

Notice that none of the numbers have quotation marks around them. If you wrote "42", the computer would treat it as a word, not a mathematical value. By learning how %d and {} operate now, you are building the foundation required to display complex data readouts from the ship's sensors later.
