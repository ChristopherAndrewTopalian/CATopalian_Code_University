Universal Language Repository: Objects and Structs

In our previous lessons, we stored individual pieces of data and lists of identical data. But what if we want to represent a complete Entity? What if we need to store a starship's name (String) AND its shield percentage (Integer) together in a single unit?

In scripting languages, this is called an Object, Dictionary, or Table. In compiled system languages, this is called a Struct (a Structure) or a Class.

Crucial Operational Note for Compiled Languages: When you compile C, C++, C#, or Rust into an executable (.exe) file and double-click it, the terminal will flash and close instantly once the code finishes. To prevent this, we add an input command at the very bottom of those scripts to hold the console window open until you press Enter!

JavaScript (HTML / Browser)

JavaScript uses curly brackets {} to create an Object. We assign properties using key-value pairs and access them using dot notation. 

<!DOCTYPE html>
<html>
<head>
    <script>
        let ship = { name: "Enterprise", shields: 100 };
        console.log(ship.name + " shields at " + ship.shields + "%");
    </script>
</head>
<body>
</body>
</html>

JavaScript (Node.js)

The Node.js desktop engine utilizes the exact same syntax. When run via an open terminal, the console naturally stays open to display your readout. 

let ship = { name: "Enterprise", shields: 100 };
console.log(ship.name + " shields at " + ship.shields + "%");

Python

Python calls these Dictionaries. They also use curly brackets, but require strings for the keys when accessing the data via square brackets. 

ship = {"name": "Enterprise", "shields": 100}
print(ship["name"] + " shields at " + str(ship["shields"]) + "%")

Lua

Lua uses its universal building block: the Table. By assigning named keys inside the curly brackets, the table acts just like a JavaScript object, allowing clean dot notation! 

ship = {name = "Enterprise", shields = 100}
print(ship.name .. " shields at " .. ship.shields .. "%")

C

In legacy C, we must define a blueprint called a struct first, explicitly declaring the memory for the character array and the integer. Notice we add getchar() at the end to keep your terminal window open! 

#include <stdio.h>

// Step 1: Define the Blueprint
struct Starship {
    char name[50];
    int shields;
};

int main() 
{
    // Step 2: Create the entity
    struct Starship ship = {"Enterprise", 100};
    
    printf("%s shields at %d%%\n", ship.name, ship.shields);
    
    // Hold the terminal window open!
    printf("Press Enter to exit...");
    getchar();
    return 0;
}

C++

C++ makes structures clean and simple. We define our blueprint using standard std::string objects—avoiding pointers and references entirely! We use std::cin.get() at the end to hold the comm-link open. 

#include <iostream>
#include <string>

// Define a clean, pointer-free blueprint
struct Starship {
    std::string name;
    int shields;
};

int main() 
{
    Starship ship = {"Enterprise", 100};
    
    std::cout << ship.name << " shields at " << ship.shields << "%" << std::endl;
    
    // Hold the terminal window open!
    std::cout << "Press Enter to exit...";
    std::cin.get();
    return 0;
}

C#

In C#, everything lives inside a Class. We build a simple blueprint with public fields so we can easily assign our starship data. We use Console.ReadLine() at the end to prevent the window from vanishing! 

using System;

class Starship 
{
    public string name;
    public int shields;
}

class Program 
{
    static void Main() 
    {
        Starship ship = new Starship { name = "Enterprise", shields = 100 };
        
        Console.WriteLine(ship.name + " shields at " + ship.shields + "%");
        
        // Hold the terminal window open!
        Console.WriteLine("Press Enter to exit...");
        Console.ReadLine();
    }
}

Rust

Rust utilizes the struct keyword. Notice that to store text inside a struct safely, Rust explicitly uses String::from(). We add an input read command at the bottom to hold your compiled window open! 

use std::io;

struct Starship {
    name: String,
    shields: i32,
}

fn main() 
{
    let ship = Starship {
        name: String::from("Enterprise"),
        shields: 100,
    };
    
    println!("{} shields at {}%", ship.name, ship.shields);
    
    // Hold the terminal window open!
    println!("Press Enter to exit...");
    let mut buffer = String::new();
    io::stdin().read_line(&mut buffer).unwrap();
}

Engineering Summary

Scripting languages allow you to build objects on the fly by simply typing curly brackets {}. Compiled system languages (C, C++, C#, Rust) require you to design a structural blueprint (a struct or class) before you can instantiate your object in memory. This upfront blueprinting is what allows compiled languages to achieve blazing fast execution speeds!
