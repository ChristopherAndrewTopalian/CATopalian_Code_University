String Variables

We know how to transmit raw data. But if we want to build a real application, we cannot just type data directly into our print commands. We need to store that data in the computer's memory so we can use it, change it, and retrieve it later.

We do this by creating a Variable. Think of a variable as a labeled cargo bay. We give the cargo bay a name (like shipName), and we put our data (like "Enterprise") inside it. Let's look at how different languages label and store a String in memory.

JavaScript (HTML / Browser)

JavaScript uses the let keyword to create a variable. It automatically detects that the data inside the quotes is a String. 

<!DOCTYPE html>
<html>
<head>
    <script>
        let shipName = "Enterprise";
        console.log(shipName);
    </script>
</head>
<body>
</body>
</html>

JavaScript (Node.js)

The desktop environment uses the exact same let syntax for flexible memory storage. 

let shipName = "Enterprise";
console.log(shipName);

Python

Python is the most minimalist. It doesn't even require a keyword like let. You simply write the name of the variable, use the equals sign, and assign the data. 

shipName = "Enterprise"
print(shipName)

C

Because C is so old, it does not actually have a "String" variable type! To store text, we must explicitly tell the computer to build an Array of Characters using char[]. We then retrieve it using the %s (String) placeholder. 

#include <stdio.h>

int main() 
{
    // C requires an Array of Characters
    char shipName[] = "Enterprise";
    
    // %s is the placeholder for a String
    printf("%s\n", shipName);
    return 0;
}

C++

C++ fixes the legacy issues of C. By including the <string> library, we get access to the powerful and safe std::string object type. We must explicitly declare the data type before the variable name. 

#include <iostream>
#include <string> // Required for the easy String object

int main() 
{
    // Explicitly declare the type as a std::string
    std::string shipName = "Enterprise";
    
    std::cout << shipName << std::endl;
    return 0;
}

Rust

Rust combines modern flexibility with system-level power. Like JavaScript, it uses the let keyword, allowing the engine to automatically infer that the text is a String without requiring you to explicitly declare the type. 

fn main() 
{
    let shipName = "Enterprise";
    
    // The empty brackets {} act as our safe placeholder
    println!("{}", shipName);
}

Engineering Summary

This reveals the two major categories of memory management. Dynamically Typed languages (JavaScript, Python, Rust) figure out what the data is automatically based on what you assign to it. Statically Typed languages (C, C++) require you to explicitly write the data type (like char[] or std::string) before the variable name so the computer knows exactly how much RAM to reserve.
